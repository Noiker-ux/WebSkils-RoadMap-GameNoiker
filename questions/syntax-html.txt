В каком регистре лучше писать HTML-код?
--
В верхнем 
В зависимости от HTML тега
В нижнем 
Без разницы
+++++
В нижнем 
--
Раньше считалось, что это безразлично. 
Но с приходом XHTML на этот вопрос появился однозначный ответ - все теги, атрибуты и предопределенные значения пишем в нижнем регистре.









Почему каждый HTML документ начинается с тега-декларации <!DOCTYPE html>
--
Элемент <!DOCTYPE> предназначен для конвертации тегов, в визуальное изображение сайта
Элемент <!DOCTYPE> подключает браузерный движок, без <!DOCTYPE> невозможно начать работу
Элемент <!DOCTYPE> предназначен для указания типа текущего документа
Элемент <!DOCTYPE> позволяет видеть страничку одинаково, вне зависимости с какого браузера она была открыта
++
Элемент <!DOCTYPE> предназначен для указания типа текущего документа
--
Таким образом браузер понимает, что он читает именно HTML










Расшифровка HTML
--
HyperText Markup Language - «Язык гипертекстовой разметки»
HyperTags Markup Language - «Язык разметки гипертегов»
HyperText Master Language - «Основной язык гипертекста»
HyperTags Master Language - «Основной язык гипертегов»
--
HyperText Markup Language - «Язык гипертекстовой разметки»
--
HTML (HyperText Markup Language) расшифровывается как язык гипертекстовой разметки. 
HTML — это стандартный язык форматирования текста, который создает и отображает различные страницы в Интернете.










Все ли HTML-теги применяются попарно?
--
Да
Нет
++
Нет
--
Нет. HTML-теги не всегда применяются парами, так как некоторые из них не нуждаются в закрывающем теге. 
Например, для тега <img> закрывающие теги не требуются.










Что такое entities?
--
Это части текста ("строки"), которые являются тегами в HTML
Это части текста ("строки"), которые начинаются с символа (#)
Это части текста ("строки"), которые начинаются с символа (&) и заканчиваются точкой с запятой (;)
Это части текста ("строки"), которая находится внутри тега <xmp>
++++
Это части текста ("строки"), которые начинаются с символа (&) и заканчиваются точкой с запятой (;)
--
Entities - это комбинации знака & и буквенного или цифрового кода после нее, предназначенные для замещения символов, которые не могут встречаться в "чистом" виде в HTML-тексте,
например, символа "<".
